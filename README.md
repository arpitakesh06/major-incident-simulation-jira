# Major Incident Management Simulation — Jira Service Management

End-to-end simulation of 5 P1/P2 major incidents managed in 
Jira Service Management, aligned to ITIL 4 incident management 
best practices.

---

## Incidents Simulated

| Ticket | Title | Priority | MTTR |
|--------|-------|----------|------|
| SUP-1 | Customer Portal Application Outage | P1 | 47 mins |
| SUP-2 | Payment Service Database Connection Failure | P1 | 72 mins |
| SUP-3 | Internal HR System Server Unreachable | P2 | 90 mins |
| SUP-4 | Login Portal Authentication Service Failure | P1 | 45 mins |
| SUP-5 | File Storage Service Degradation | P2 | 90 mins |

---

## Lifecycle Followed

For each incident:
1. Incident detected and logged in Jira Service Management
2. Priority and impact assessed (Critical/High + Extensive/Widespread)
3. Bridge call opened, resolver group engaged
4. Real-time stakeholder updates drafted
5. Root cause identified and resolution applied
6. RCA documented using 5 Whys methodology
7. Problem record raised to prevent recurrence
8. Post-incident review conducted

---

## Tools Used

- Jira Service Management (Free tier)
- ITIL 4 Incident Management framework
- 5 Whys RCA methodology
- Atlassian PIR format
